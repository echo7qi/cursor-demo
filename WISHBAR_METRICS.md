# 祈愿 bar 访问贡献看板（wishbar）— 字段与计算口径

> 供业务复查。代码入口：`wishbar.js` 中 `buildWeeklyShares`、`getFilteredRows`、`getFilteredParsed`、`aggregateTrendUnionSeries`、`buildWishbarContributionRows`、`buildWishbarWeeklyReportHtml`。

---

## 1. 原始数据与行级规则

| 列（逻辑名） | 含义 | 聚合方式 |
|-------------|------|----------|
| **日期** | 周标识，与 CSV 字符串**完全一致**（如 `2026-03-16`） | 作为周键 |
| **统计周期** | 若表头存在该列且**至少一行非空**：仅保留值含「周」的行；若整列都空：**不**按周期过滤（所有行进入聚合，存在与日维度混用风险） | 过滤 |
| **一级来源** | 业务一级来源 | 空/`--`/nan → 规范为 **`(空)`**；`广告资源投放`/`v2资源投放` 等 → **`运营资源投放`** |
| **二级来源** | 业务二级来源 | 同上 `normSource`（展示层模块三已不用二级筛选） |
| **祈愿 bar 访问 uv** | 访问用户量 | `num()`：去逗号转数字，非法→0 |
| **抽卡用户数** | 抽卡用户 | 同上 |
| **付费抽卡用户数** | 付费抽卡用户 | 同上 |
| **付费金额** | 金额 | 同上 |
| **活动名称【修正】** | 项目 / 活动维度（简报 Top5 用） | 见下文「项目付费 Top5」 |

- **行是否进入某周聚合**：通过周期过滤后，该周键下四个指标**全为 0** 的行会跳过（不累加）。
- **重复行**：同一周、同一一级来源多行会 **相加**（若上游为明细需自行去重）。

## 2. 周聚合（`buildWeeklyShares`）

对每个「周」、每个「一级来源」：

- **访问 UV / 抽卡用户 / 付费抽卡用户 / 付费金额** = 符合规则的所有行对应字段 **求和**。
- **`(空)` 一级**：仍参与 **分母**（全周 total），模块一/二/四 **展示时隐藏** `(空)` 行或列；**简报与合计仍含 `(空)` 流量**。

### 全周合计 `totalsByWeek(w)`

- `uv` = 当周所有一级来源 UV 之和（含 `(空)`）
- `draw` / `payUser` / `payAmount` 同理

### 一级来源占比（模块二、模块四）

对固定周、每个非空一级 `s`（`l1All` 为历史出现过的全部一级）：

- **访问占比** = `该来源当周 UV / 当周 totalsByWeek.uv × 100%`（分母为 0 则为 0）
- **抽卡用户数占比** = `该来源 draw / 当周 totalsByWeek.draw × 100%`
- **付费抽卡用户数占比** = `该来源 payUser / 当周 totalsByWeek.payUser × 100%`
- **付费金额占比** = `该来源 payAmount / 当周 totalsByWeek.payAmount × 100%`

> 各占比的分母 **互不混用**（访问用访问总量，付费用户用付费用户总量等）。

### 二级来源占比 `l2SharesByWeek`

- 分母为当周二级 UV 合计；当前模块三未使用。

## 3. 全局筛选（`getFilteredParsed`）

当 **开始/结束周、一级、二级** 任一非「全选」时：

- 对 **`rawRows`** 做 `getFilteredRows`：  
  - 周字符串与起止 **字典序比较**（`>=` `<=`），须与 CSV 周格式一致。  
  - 一级/二级：多选为 **AND**：必须同时命中（在已选集合内）。
- 再 **`buildWeeklyShares(过滤后行)`** 全量重算周聚合与占比。  
- **不会**使用未调用的 `applyGlobalFilters`（已标 deprecated，避免只切片 map 不重算分母）。

---

## 4. 模块三趋势：并集汇总线（`aggregateTrendUnionSeries`）

- **原始行**：与全局筛选一致（`getRowsForTrendAggregate`）。
- **横轴周**：`fp` 经模块三起止周截取后的 `fpTrend.weeks`。
- **命中行**：一级属于所选并集（支持弹窗里的 **「其他」** 桶：`cleanL1` 以外的一级）；模块三默认 **全选非空一级** = 命中当周**几乎全部**行。
- **分子**（每周）：命中行的 UV / draw / payUser / payAmount **分别求和**。
- **分母**（每周）：`fp.totalsByWeek` 的 **全周** 对应指标（与 `buildWeeklyShares` 一致，**含 `(空)`**）。
- **四条线含义**：  
  - 访问占比 = `分子 UV / 分母 uv × 100%`  
  - 抽卡用户数占比 = `分子 draw / 分母 draw × 100%`  
  - 付费抽卡用户占比、付费金额占比同理。

**默认全选一级时**：分子≈分母 → 四条线约 **100%**（正常，表示全站周汇总）。

**指数纵轴**：在截取后的周序列上，对每条占比序列做 **首周 = 100** 的指数化，仅便于看形态；tooltip 仍带真实占比与汇总。

---

## 5. 模块二「千次访问」

对每个一级来源、选定周：

- **千次访问·抽卡用户** = `draw / uv × 1000`（`uv≤0` 显示 —）
- **千次访问·付费抽卡用户** = `payUser / uv × 1000`
- **千次访问·付费金额** = `payAmount / uv × 1000`

简报里「千次排名」仅 **UV ≥ 1000** 的来源参与，降低小样本波动。

## 6. 简报：当周付费贡献 Top5（项目 / 活动）

- **数据源**：与全局筛选一致的 **原始行** `getRowsForTrendAggregate()`，**不**经过一级来源聚合。
- **分组键**：表头识别 **`活动名称【修正】`**（或别名：活动名称、项目名等，见 `COLS_ALIASES.activity`）。
- **纳入当周**：`日期` 列等于所选周，且统计周期规则与 `buildWeeklyShares` 一致（有周期列则须含「周」）。
- **聚合**：同名活动（去空格后空名 → `(未命名活动)`）对 **付费抽卡用户数**、**付费金额** 分别 **求和**；两指标均为 0 的行不参与累加。
- **排除**：活动列 trim 后等于 **「整体」** 的行视为汇总行，**不参与** Top5 聚合与排序（避免与分项活动重复占榜）。
- **综合分（用于排序）**：  
  - \(u_i\) = 活动 \(i\) 的付费用户数，\(a_i\) = 付费金额；  
  - \(U=\max_i u_i\)，\(A=\max_i a_i\)（若为 0 则分母取 1 避免除零）；  
  - **score\(_i\)** = \(0.3 \times (u_i/U) + 0.7 \times (a_i/A)\)（**付费用户 30% + 付费金额 70%**）。  
  按 score 降序取前 5。分数仅 **周内可比**，非业务绝对 KPI。

## 7. 周度简报（环比 / 季度）

- **简报 KPI 四卡**：主数字为当周全站合计；括号内为 **周环比**（相对 `fp.weeks` 中上一数据周，与下方「环比上周」表同一口径）；上周为 0 或无更早一周时显示 `（—）`。
- **KPI 归因（一级来源）**：当全站环比 **非持平**（|环比|≥0.05%）时，在卡片底部提示 **主因一级来源**：在变动方向与全站一致的一级来源中，取 **本周−上周 绝对增量最大** 者；展示其占全站变动的比例（`Δ来源/Δ全站`，仅当同向且 ≤100%）及该来源自身环比。一级 **(空)** 不参与候选（与模块二展示口径一致）；若全站变动为 0 则不展示。
- **来源 Top3**：按 **访问占比** 排序后的前三个一级来源（与下方「付费结构快照」分工：前者看流量集中，后者看付费/千次榜首）。
- **付费结构快照**：列出 **付费用户占比**、**付费金额占比**、**千次付费额**（UV≥1000）各自第一的一级来源；若与访问体量第一不一致，附一句对照说明。
- **环比上周**：在 **`fp.weeks` 排序数组** 中取选定周的 **上一元素**（非日历推断；**缺周则对比的是「数据里的上一行」**）。
- **总量环比**：`(本周 − 上周) / 上周 × 100%`；上周为 0 → 不展示有效环比。
- **访问占比第一名的百分点变化**：本周占比 − 上周**同名来源**占比（百分点，非相对变化率）。
- **季度趋势**：周标签能解析为 `YYYY-MM-DD` 时按 **自然季** 筛周；对 **全周 UV 序列** 做一元线性回归斜率；**非预测承诺**，仅描述历史斜率方向。

## 8. 已知限制（自查结论）

1. **周序**：`fp.weeks` 为周键 **字符串排序**，非严格日历序（`YYYY-MM-DD` 下通常正确）。
2. **统计周期列全空**：不按「周」过滤，可能混入非周粒度行。
3. **`applyGlobalFilters`**：未使用；勿改为只切片 map 而不 `buildWeeklyShares`。
4. **模块三默认全选一级**：曲线为全站汇总（约 100%）；要看得分结构请在「一级来源」里改选子集。
